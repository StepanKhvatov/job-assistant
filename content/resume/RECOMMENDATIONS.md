## 3. Best practices 2026 (что уже учтено в master.md)

1. **Длина — 1 страница.** Сейчас укладывается.
2. **Конкретика и масштаб** — 3000+ страниц, 1500+ объектов, команда 3 фронта, сборка 1–3 мин.
3. **Ключевые слова в первых 2/3 текста** — App Router, RSC, Server Actions, Nx, SEO, Core Web Vitals.
4. **Параллельная занятость подписана** — full / part. На hh поможет избежать вопросов про пересечения.
5. **Gap не маскируем** — статус «открыт к предложениям» вынесен в шапку.
6. **Дубли убраны** — React.js, HTML/CSS как отдельные строки, «(базово)» удалены.

### Чего избегать
- Общих фраз «ответственный, стрессоустойчивый».
- Стека из 30+ навыков без приоритета.
- Описания обязанностей вместо результатов («разрабатывал, тестировал»).
- Скрывать SquareFi из-за конфликта с тимлидом — 4 месяца на крипто-проекте с SSA это сильный пункт, не red flag.

## 4. Сильные истории для собеседования (storytelling)

При подготовке к интервью держите наготове 3 рассказа по схеме **Situation → Action → Result**:

### Story 1 — Server-side auth в SquareFi (15 мин, технический)
- **Situation:** нужно сделать JWT-авторизацию в Next.js 15 для Web + Telegram Mini App, без `next-auth` (по требованию команды).
- **Action:** access/refresh в httpOnly cookies; auth-middleware вокруг OpenAPI-клиента; при 401 — refresh с дедупом через Mutex + AsyncLocalStorage + React `cache()`; sync куков из RSC через клиентский компонент.
- **Result:** один refresh на параллельные 401-запросы; работают и Server Components, и Server Actions, и Route Handlers; покрыто E2E на Playwright. **Рефлексия:** если бы делал заново — взял бы готовый Auth.js v5, реализация заняла больше времени, чем планировал.

### Story 2 — Nx monorepo и SEO в Immigrant Invest (10 мин, продуктовый)
- **Situation:** 3 продукта в одном репозитории, плюс запустили дочерний мультитенант (imin-cyprus) ради SEO; общие компоненты, общая CMS.
- **Action:** организовали Nx workspace, общие UI-блоки (Storyblok-driven), мультиязычные роуты через next-intl, оптимизация LCP/CLS на тысячах страниц.
- **Result:** деплой 1–3 мин, 2–3 релиза в неделю; запустили новый бренд (imin-cyprus) переиспользовав 80% кода.

### Story 3 — миграция Strapi 3 → 4 (5 мин, бэк-составляющая)
- **Situation:** на Immigrant Invest пришла major-миграция Strapi с ломающими изменениями API и схемы.
- **Action:** написал миграционные скрипты для контента (объекты недвижимости 1500+), обновил контроллеры/ручки, поправил frontend-вызовы.
- **Result:** мигрировали без потери данных; параллельно начали использовать Strapi Cloud.

### Bonus — pet `job-assistant`
- Расскажите про этот репозиторий на скрининге как пример проактивности: «написал ассистента, который сам скрапит hh, ранжирует вакансии через DeepSeek и автоматически откликается». Сильный сигнал на product-mindset.

## 8. Ритм поиска

- **Откликов в день:** 20–40 (закрывает `job-assistant`).
- **Целевая конверсия:** 5–10% откликов → скрининг, 30–50% скринингов → техсобес, 30–40% техсобесов → оффер.
- **Прогноз:** первый оффер за 3–8 недель, хороший оффер (250k+) за 6–10 недель.

### Главный риск
Не «не возьмут», а **«возьмут не туда»**: согласиться на первый оффер на 200k, когда через 2 недели мог быть 320k.

**Правило:** не принимать офферы до 4-й недели поиска — собрать 3–5 параллельных предложений и сравнить.
