# Сайт-визитка khvatov.ru — план улучшений

Контекст: текущий сайт [khvatov.ru](https://www.khvatov.ru/) выглядит скромно (одна шапка с именем + страница [resume](https://www.khvatov.ru/resume)), нужно добавить проекты. Это **отдельный репозиторий** (не `job-assistant`); данный файл — справочник, что и в каком порядке вносить там.

Источник правды по проектам и стеку: [`content/resume/master.md`](../content/resume/master.md).

---

## 1. Цель сайта

Сайт-визитка нужен для **2 аудиторий** в порядке приоритета:

1. **Рекрутер / тимлид**, который перешёл по ссылке из hh/LinkedIn. Сценарий: 30 секунд скана, ищет «что человек делал руками + чем выделяется».
2. **Случайный коллега / гость**, который ищет ваши контакты или работы.

Оба сценария решаются **одной страницей** с якорями: hero → projects → skills → contact. Отдельная вкладка `/resume` остаётся (и обновляется по master.md), но главная — главный путь.

## 6. Технические рекомендации

### SEO
- `<title>` на главной: `Stepan Khvatov — Middle Frontend Developer (React, Next.js)`.
- `<meta name="description">`: 1 предложение под §5.
- `og:image` — сгенерированный (next/og) баннер с именем и ролью.
- `sitemap.xml` + `robots.txt` (Next.js генерирует автоматически).
- `hreflang` на `ru` и `en`.

### Performance
- Lighthouse цель: **95+ по всем 4 метрикам**.
- Изображения проектов — `next/image` с предзагрузкой LCP-картинки.
- Шрифты — `next/font` (вы уже используете Inter).
- Никакого React Server Component-bloat: hero/cards = простые server-компоненты, без лишнего JS.

### Доступность
- Контрастность по WCAG AA (тёмная тема ок, но проверить серый текст).
- Все интерактивные элементы — нативные `<a>` / `<button>`, фокус видимый.
- `aria-label` на иконочных ссылках в подвале (у вас уже есть).

### Аналитика (опционально)
- Plausible или Umami — минималистично и без cookie banner.
- Цель в аналитике — клики по «GitHub», «CV», «LinkedIn».

## 7. Чего не делать

- **Не делайте «3D-портфолио» с анимациями.** Рекрутеру нужна скорость, а не WOW.
- **Не пишите длинных описаний.** Карточка проекта = ≤ 60 слов.
- **Не дублируйте `/resume`** на главной. На главной — карточки проектов; полное резюме — отдельная страница.
- **Не оставляйте «настоящее время» у проектов, которые закончились.** На вашем `/resume` сейчас «настоящее время» у DexPilot и Flake — это устарело.
- **Не выкладывайте сырой PDF резюме** с телефоном и email — пусть будет вариант без ПДн или контакты только в кнопках.
